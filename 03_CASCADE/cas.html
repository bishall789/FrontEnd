CASCADE , SPECIFITY AND INHERITENCE

CASCADE

Top to Bottom selector priority increase (top have least & bottom have highest priority.

The word "Cascading" in CSS means that when multiple styles apply to the same element, CSS follows a set of rules to decide which one should take effect.

1️⃣ Order of Importance (What Comes First?)

CSS styles come from three main sources, and they are applied in this order:

1. Inline Style
They have highest priority and can override Internal and External style

2. Internal Style
They have medium priority and can override External style

3. External Style
They have lowest priority and can be overridden by Inline and Internal style

CASCADING -> In same style, Ex : External Style previous rules are overwritten by newer rules.

As we go down to style sheet, its priority keeps increasing

SPECIFITY (Who Wins in a Conflict?)

If multiple rules target the same element, CSS uses a specificity ranking to determine the winner:

  • Inline styles are the strongest.
  • ID selectors (#id) are stronger than class selectors.
  • Class selectors (.class) are stronger than element selectors (h1, p).
  • If two rules have the same strength, the last one written wins.
  • !important Overrides Everything
  1. Inline Styles → 1000
  2. ID Selectors (#id) → 100
  3. Class , Attribute, Pseudo-class Selectors (.class, [attr], :hover) → 10
  4. Element Selectors (p, h1, div) → 1
  5. Universal Selector (*) → 0
  6. !important → Overrides all specificity

4️⃣ Final Rule Application (How CSS Decides the Style)

  • CSS checks for !important rules (they override everything).
  • If no !important, CSS applies the most specific rule (ID > Class > Element).
  • If multiple rules have the same specificity, the last one written wins.
